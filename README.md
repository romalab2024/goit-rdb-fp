# 📊 Pandemic Data Analysis (SQL Final Project)

## 📌 Опис проєкту

У цьому проєкті реалізовано аналіз даних про інфекційні захворювання з використанням SQL. Основна мета — імпорт, нормалізація та подальша аналітика даних у реляційній базі.

---

## 🧱 Структура проєкту

### 1. Створення схеми
Створено окрему схему `pandemic` для роботи з даними.

### 2. Імпорт даних
Дані імпортовано з CSV-файлу у таблицю `infectious_cases`.

---

## 🧩 Нормалізація (3NF)

Дані приведено до третьої нормальної форми:

- Таблиця `entities` (довідник країн)
- Таблиця `infectious_cases_normalized` (основні дані)

Це дозволяє:
- уникнути дублювання
- підвищити цілісність даних
- покращити масштабованість

---

## 📈 Аналіз даних

Було виконано агрегацію для показника `Number_rabies`:

- середнє значення (AVG)
- мінімальне значення (MIN)
- максимальне значення (MAX)
- сумарне значення (SUM)

З урахуванням очищення даних:
- виключення порожніх значень
- приведення типів до числових

---

## 📅 Робота з датами

### Task 4. Додавання колонок із датами

Для виконання завдання було використано команди `ALTER TABLE` та `UPDATE`.

У таблицю `infectious_cases_normalized` додано три нові колонки:

- `year_start_date` — дата першого січня відповідного року;
- `current_date_value` — поточна дата;
- `year_difference` — різниця в роках між поточною датою та датою, створеною з колонки `Year`.

Дані в нові колонки були записані за допомогою `UPDATE` з використанням функцій `STR_TO_DATE`, `CONCAT`, `CURDATE` та `TIMESTAMPDIFF`.
